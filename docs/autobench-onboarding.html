Autobench: getting started

Autobench compares one entity (an issuer, bank, or merchant) against its peers. You give it a CSV of pre-aggregated data; it gives you an Excel report that automatically satisfies privacy rules.

This page gets you from zero to one successful run. Total time: about 10 minutes.

What it does

One paragraph of theory, then we run things.

A naive peer average is easy to skew: one very large peer can dominate the whole comparison. Autobench reweights the peer group so no single peer exceeds a concentration cap, then compares your target entity against that balanced group. The caps come from privacy rules and are enforced automatically — you never configure them in normal use.

There are two analysis modes:

ModeQuestion it answersExample
Share How is my activity distributed across categories, compared to peers? % of transactions by channel, card type, or geography
Rate How does my performance rate compare to peers? Approval rate, fraud rate

There are four ways to run it — a guided terminal UI (TUI), a command-line interface (CLI), preinstalled launchers on Edge Nodes, and a Python API. They all use the same engine and produce the same results. If you are new, start with the TUI.

Before you start

You need one of these two setups.

Working from the git repository

  • Python 3.10 or newer.
  • A clone of this repository.
  • Install dependencies once, from the repository root:
python -m pip install -e ".[dev]"

Working on a deployed Edge Node

  • Nothing to install. Do not run pip.
  • Run the onboarding script once (details in the Edge Node tab below).

In this repository, Python commands are written as py .... If py does not exist on your system, use python instead.

Run the demo

Do this before touching your own data. The repository ships a known-good demo file, tests/fixtures/gate_demo.csv (1 target + 6 peers), so you can learn the workflow without debugging your data at the same time.

The TUI shows every choice on screen. Best way to learn what the options mean.

  1. From the repository root, launch it: 
    py tui_app.py
  2. In CSV File Path, type tests/fixtures/gate_demo.csv, then press Enter (or Tab away) so the column headers load.
  3. Set Entity ID Column to issuer_name and Target Entity to Target (case-sensitive).
  4. On the Share tab: metric txn_cnt, time column year_month, dimensions card_type and channel, preset balanced_default.
  5. Click Run Analysis. The log should end with Analysis completed successfully and a saved report path.

Same run, one command. Run from the repository root.

Share analysis

py benchmark.py share \
  --csv tests/fixtures/gate_demo.csv \
  --entity Target \
  --metric txn_cnt \
  --dimensions card_type channel \
  --time-col year_month \
  --preset balanced_default \
  --output gate_demo_share.xlsx

Rate analysis (also exports a balanced CSV)

py benchmark.py rate \
  --csv tests/fixtures/gate_demo.csv \
  --entity Target \
  --total-col total \
  --approved-col approved \
  --dimensions card_type channel \
  --time-col year_month \
  --preset balanced_default \
  --export-balanced-csv \
  --output gate_demo_rate.xlsx

On Windows PowerShell, replace the trailing \ with ` and use backslashes in paths (tests\fixtures\gate_demo.csv).

Edge Nodes have a shared runtime managed by the Release Operator. You never install anything.

  1. Onboard once (also safe to rerun any time your launchers break): 
    cd /ads_storage/autobench
./onboard.sh
  2. Go to the directory with your data and launch: 
    cd /path/to/my/work
autobench            # the TUI
autobench-cli share --help   # the CLI
  3. If the commands are not found: 
    export PATH="$HOME/.local/bin:$PATH"
which autobench
which autobench-cli

If onboarding says the shared runtime is missing or invalid, stop and send the exact error to the Release Operator. Do not run install.sh or pip yourself.

Check it worked

All four must be true. If any is false, go to Troubleshooting.

  • The command exited with code 0 (CLI) or the TUI log says Analysis completed successfully.
  • The output file exists (e.g. gate_demo_share.xlsx; the rate run also writes gate_demo_rate_balanced.csv).
  • The Summary sheet in the workbook shows Input Validation: pass.
  • The Summary sheet shows Compliance Verdict: fully_compliant.

The generated .xlsx, .csv, and benchmark_log_*.txt files are local artifacts. They are gitignored and safe to delete.

Use your own data

Once the demo works, swap in your CSV. Change one thing at a time: first the file, then the entity and metric, then the dimensions. Keep balanced_default.

Your CSV must be pre-aggregated: one row per entity × dimension bucket (optionally × time period). Like this:

issuer_name,year_month,card_type,channel,txn_cnt,total,approved,fraud
Target,2026-01,CREDIT,POS,120000,120000,112800,84
Peer A,2026-01,CREDIT,POS,98000,98000,91630,71
Peer B,2026-01,CREDIT,POS,105000,105000,98700,76

Rules that cause most first failures:

  • Entity names are case-sensitive. Target and target are different entities.
  • Column names get normalized to lowercase with underscores. Refer to them that way.
  • Units must be consistent across all rows (same currency, same count definition).
  • You need at least 5 peers for a normal (non-merchant) run. Fewer peers means the privacy rule cannot be satisfied.
  • No nulls in the entity or metric columns.

If validation fails, fix the data — do not disable validation to force a run. A workbook produced from bad input is not a usable benchmark.

Privacy rules

Automatic. Listed here so you know what the tool is doing, not so you can tune it.

The engine picks the applicable concentration rule based on peer count and enforces it. The baseline is 5/25: at least 5 peers, and no peer contributes more than 25% to any category-level comparison.

RuleMin peersMax concentrationNotes
5/25525%The normal starting rule
6/30630%Requires ≥3 participants at ≥7%
7/35735%Requires ≥2 at ≥15% and ≥1 at ≥8%
10/401040%Requires ≥2 at ≥20% and ≥1 at ≥10%
4/35435%Merchant benchmarking only

Some outputs, such as top-merchant lists, are blocked outright. Do not restructure a request or disable checks to get around a block — escalate the use case to Privacy. Special contexts (digital-wallet metrics, recurring deliverables, altered peer groups) need explicit declarations or review evidence.

Read the output

Open the workbook. Check Summary first, then read the dimension sheets.

Sheet / fileWhat it tells you
SummaryRun inputs, preset, validation result, and compliance verdict. Always check this first.
One sheet per dimensionTarget vs. balanced peers per category, with gaps and best-in-class comparisons. This is the answer to your business question.
Weight MethodsWhich weighting strategy each dimension used (Global-LP, Per-Dimension-LP, or Bayesian fallback).
Rank ChangesHow the reweighting shifted peer ranks — a distortion check.
Balanced CSVBalanced metrics for BI tools (Power BI, Tableau) or pipelines. Written with --export-balanced-csv.
Audit packageOne zip with workbook(s), CSV, audit log, config snapshot, and validation summary. Written with --audit-package.

The analysis workbook is for internal work. If the result needs to leave the analysis room, produce a publication workbook with --output-format publication (or both) — it goes through sanitization and publishability validation. JSON sidecars are analysis-grade, not publication-redacted.

Troubleshooting

Fix the first concrete error. Most first-run failures are an exact-value mismatch.

“Entity not found”

The value of --entity must match the CSV exactly, including case. In the demo file the target is Target, capital T.

“Column not found”

Column names are normalized to lowercase with underscores (Card Type becomes card_type). If unsure, load the file in the TUI and look at the headers it lists.

“No valid dimensions”

Pass dimensions explicitly, e.g. --dimensions card_type channel. Only use --auto once explicit dimensions work.

LP infeasibility warnings or high distortion

Some sparse datasets cannot satisfy every constraint; the engine falls back to a weaker method and reports it in Weight Methods. Use --compare-presets and --analyze-impact to see the trade-off. This is a data property, not a bug.

Edge Node: shared runtime missing or invalid

Rerun the onboarding script:

/ads_storage/autobench/onboard.sh

If it still fails, send the exact error to the Release Operator. Analysts cannot repair or replace the shared runtime.
