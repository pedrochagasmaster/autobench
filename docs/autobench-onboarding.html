Autobench · onboarding & first use

Trust the benchmark, not the loudest peer.

Autobench compares an entity with its peers while automatically limiting peer concentration. You bring pre-aggregated data; it returns privacy-compliant, decision-ready benchmarks.

01 · Mental model

What Autobench actually does

Think of it as a controlled comparison pipeline. It checks the input, creates a representative peer benchmark, tests privacy constraints, and packages evidence alongside the result.

A

Your target stays visible

Choose one entity to compare—or omit it for a peer-only view. Entity names are case-sensitive.

B

Peers are reweighted

Autobench balances the peer group so no single participant dominates a category-level comparison.

C

Evidence travels with the result

Reports expose inputs, weight methods, rank changes, compliance verdicts, and optional audit artifacts.

Input

Pre-aggregated CSV

One row per entity × dimension bucket, with consistent metric units.

Validate

Schema & data checks

Columns normalize; missing keys, invalid values, and unsafe shapes surface early.

Balance

Peer weighting

The engine chooses a feasible weighting strategy for the selected dimensions.

Protect

Privacy enforcement

Control 3 rules cap concentration and block disallowed deliverables.

Deliver

Workbook & exports

Analysis, publication, CSV, JSON, diagnostics, or an audit package.

→
The practical “so what”

Differences in the output are less likely to be artifacts of one oversized peer—and easier to explain, inspect, and reproduce.

02 · Features & use cases

Choose the question, then the surface

The TUI, CLI, and Python API use the same core engine. Choose based on how often the work runs and who needs the result.

Share

Where does activity happen?

Compare how a target’s transaction count, amount, or another primary metric is distributed across categories.

Best for: channel mix, card type, geography, product mix.
Rate

How does performance differ?

Benchmark approval or fraud rates using a total denominator and one or more numerator columns.

Best for: approval performance, fraud incidence, operational rates.
Peer-only

What does the peer market look like?

Omit the target entity to produce a balanced peer reference without a target-vs-peer comparison.

Best for: neutral baselines and market context.
Publication

What can leave the analysis room?

Create a publication-oriented workbook after output sanitization and publishability validation.

Best for: controlled downstream sharing.
BI export

How do I feed a dashboard?

Export balanced CSV data for Power BI, Tableau, or a data pipeline, with export validation enabled by default.

Best for: repeatable dashboards and downstream analysis.
Automation

How do I schedule this?

Use the stable Python API for notebooks and pipelines, or the CLI for simple job orchestration.

Best for: scheduled runs and embedded workflows.
i
New here? Start with the TUI and balanced_default.

It makes the required choices visible. Once the run makes sense, copy the same contract into a CLI command or AnalysisRunRequest.

03 · Prepare

Bring data the engine can reason about

A clean first run begins before the command. Confirm these six things about the input.

  • Pre-aggregated rowsOne row per entity × dimension bucket, optionally × time period.
  • Stable entity keyDefault column is issuer_name; values are case-sensitive.
  • Primary metricA comparable count, amount, or denominator with consistent units.
  • Useful dimensionsColumns such as card_type and channel.
  • Enough peersNormal non-merchant analysis starts at five peers for the 5/25 rule.
  • No sensitive output designDo not use Autobench to produce top-merchant lists or bypass privacy review.
issuer_name,year_month,card_type,channel,txn_cnt,total,approved,fraud
Target,2026-01,CREDIT,POS,120000,120000,112800,84
Peer A,2026-01,CREDIT,POS,98000,98000,91630,71
Peer B,2026-01,CREDIT,POS,105000,105000,98700,76
!
Do not disable validation just to make a failing dataset run.

Fix the schema or data issue first. A generated workbook is not proof that the comparison is safe or meaningful.

04 · First use

Complete one known-good run

Use the tracked gate_demo.csv fixture. It contains one target and six peers, so you can learn the workflow without diagnosing your own data at the same time.

See every choice on screen

Best for a first encounter with the entity, metric, dimension, and preset concepts.

  1. Install the contributor environment

    python -m pip install -e ".[dev]"

  2. Launch from the repository root

    py tui_app.py

  3. Load the fixture

    Enter tests/fixtures/gate_demo.csv, then press Enter or tab away so column headers load.

  4. Configure Share analysis

    Entity column: issuer_name · Target: Target · Metric: txn_cnt · Time: year_month · Dimensions: card_type + channel · Preset: balanced_default.

  5. Run and inspect

    Click Run Analysis. The log should say Analysis completed successfully and show the saved report path.

Run the same contract from PowerShell

Use explicit dimensions for a predictable first run.

  1. Share analysis

    py benchmark.py share `
  --csv tests\fixtures\gate_demo.csv `
  --entity Target `
  --metric txn_cnt `
  --dimensions card_type channel `
  --time-col year_month `
  --preset balanced_default `
  --output gate_demo_share.xlsx

  2. Rate analysis with balanced CSV

    py benchmark.py rate `
  --csv tests\fixtures\gate_demo.csv `
  --entity Target `
  --total-col total `
  --approved-col approved `
  --dimensions card_type channel `
  --time-col year_month `
  --preset balanced_default `
  --export-balanced-csv `
  --output gate_demo_rate.xlsx

  3. Confirm the shell result

    $LASTEXITCODE
Get-Item gate_demo_share.xlsx, gate_demo_rate.xlsx, gate_demo_rate_balanced.csv

Embed Autobench without shelling out

The stable public surface is AnalysisRunRequest plus execute_share_run or execute_rate_run.

import logging
from core.analysis_run import execute_share_run
from core.contracts import AnalysisRunRequest

request = AnalysisRunRequest(
    csv="tests/fixtures/gate_demo.csv",
    entity="Target",
    metric="txn_cnt",
    dimensions=["card_type", "channel"],
    time_col="year_month",
    preset="balanced_default",
    compliance_posture="strict",
    output="report.xlsx",
)

artifacts = execute_share_run(request, logging.getLogger("pipeline"))
print(artifacts.analysis_output_file)

05 · Privacy guardrails

Privacy is a run condition, not a formatting option

Autobench selects and enforces the applicable concentration rule. Normal runs do not ask analysts to tune the caps.

“A plausible number is not automatically a publishable number.”

Autobench operating principle
5/25

The normal starting rule

At least five peers; no peer may exceed 25% concentration. Other rules apply as participant count and context change.

5/255 peers · 25% cap
6/306 peers · 30% cap
7/357 peers · 35% cap
10/4010 peers · 40% cap
4/35Merchant mode only
Automatic

Safe defaults stay on

Input validation, privacy-cap enforcement, export cross-validation, and publishable-output validation are part of the workflow.

Declare

Special contexts need evidence

Digital-wallet metrics, dual protected entity axes, recurring deliverables, and altered peer groups require explicit declarations or review evidence.

×
Top-merchant outputs are blocked.

Do not restructure the request, disable checks, or reach into internal modules to bypass a blocked deliverable. Escalate the use case to Privacy.

06 · Outputs

Read the evidence before the insight

Start in Summary, confirm validation and compliance, then interpret the dimension sheets.

Output What it tells you Use it when
Summary Run inputs, preset, validation, configuration, and compliance verdict. Always—this is the first inspection point.
Dimension sheets Target versus balanced peers, gaps, and best-in-class comparisons by category. Answering the business question.
Weight Methods Whether each dimension used Global-LP, Per-Dimension-LP, or Bayesian fallback. Explaining how the benchmark was constructed.
Rank Changes How weighting changed peer ranks and the shape of the comparison. Assessing distortion and stability.
Balanced CSV Balanced metrics for downstream BI or pipeline ingestion. Using --export-balanced-csv.
Audit package Reports, CSV, audit log, config snapshot, and validation summary in one zip. Preserving release or review evidence.
→
Analysis output and publication output are different contracts.

Use --output-format publication or both when a sanitized, publishability-validated artifact is required. JSON sidecars are analysis-grade, not publication-redacted.

07 · Troubleshooting

Fix the first concrete failure

Most first-run issues are an exact value, column, or environment mismatch.

“Entity not found”

Use the exact case-sensitive entity value from the CSV. For the demo fixture, the target is Target.

“Column not found”

Use normalized column names: lowercase with underscores. Load the file in the TUI to inspect the headers Autobench sees.

“No valid dimensions”

Pass explicit dimensions such as --dimensions card_type channel before trying --auto.

LP infeasibility or high distortion

Some sparse datasets cannot satisfy every global constraint. Inspect the fallback method and structural sheets; use --compare-presets and --analyze-impact to understand the tradeoff.

Edge Node shared runtime is missing or invalid

Rerun /ads_storage/autobench/onboard.sh. If it still fails, send the exact error to the Release Operator; analysts cannot repair or replace the shared runtime.

You are ready when…

You can explain the input, the rule, and the verdict.

After the demo succeeds, replace one concept at a time: first the CSV, then the target and metric, then the dimensions. Keep balanced_default and explicit dimensions until you have a reason to change them.
