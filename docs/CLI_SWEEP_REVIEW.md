# CLI Sweep Review Instructions

This document explains how to execute and review the CLI sweep cases generated by
`scripts/generate_cli_sweep.py`. It is aligned to `README.md` and `benchmark.py`.

## 1) Generate the sweep cases

From the repo root:

```
py scripts\generate_cli_sweep.py
```

Optional flags:

- `--csv <path>`: Force a specific CSV
- `--out-dir <path>`: Output folder (default: `test_sweeps`)
- `--mode core|exhaustive`: `core` produces a manageable matrix + feature cases
- `--max-cases N` and `--allow-large`: guardrails for `exhaustive`
- `--entity-col`, `--entity`, `--time-col`, `--metric-col`, `--total-col`, `--approved-col`, `--fraud-col`: override auto-detected columns
- `--dimensions`, `--secondary-metrics`: override inferred dimension/secondary lists

The script writes:

- `test_sweeps\meta.json` (inferred columns and entity value)
- `test_sweeps\share\cases.jsonl` + `commands.ps1`
- `test_sweeps\rate\cases.jsonl` + `commands.ps1`
- `test_sweeps\config\cases.jsonl` + `commands.ps1`
- `test_sweeps\outputs\share\` and `test_sweeps\outputs\rate\` (output target folders)

## 2) Run the cases

Use the generated PowerShell command files:

```
# Share cases
Get-Content test_sweeps\share\commands.ps1 | ForEach-Object { Invoke-Expression $_ }

# Rate cases
Get-Content test_sweeps\rate\commands.ps1 | ForEach-Object { Invoke-Expression $_ }

# Config cases
Get-Content test_sweeps\config\commands.ps1 | ForEach-Object { Invoke-Expression $_ }
```

Tip: If the dataset is large, run cases in batches. You can split the command files
or filter by case ID in `cases.jsonl`.

## 3) What to verify (global expectations)

These checks should be performed for every analysis case unless noted.

**Output files**
- `analysis` format: an analysis workbook is created.
- `publication` format: a publication workbook is created (no diagnostic sheets).
- `both`: both workbooks exist.
- Audit log: `<output_stem>_audit.log` is created (default on).
- Balanced CSV: only when `--export-balanced-csv` is set.

**Summary sheet**
- Contains inputs (CSV path, entity, metric or rate columns, dimensions).
- Shows peer count and applied privacy rule.

**Dimension sheets**
- One sheet per dimension.
- If `--entity` is provided, target columns exist (Target Share/Rate, Distance to Peer).
- If `--entity` is omitted, target columns are absent (peer-only mode).

**Data Quality sheet**
- Present when validation runs (default on unless `--no-validate-input`).
- Absent when validation is disabled.

**Balanced CSV (peer-only rule)**
- Balanced CSV must exclude the target entity and reflect peer-only totals.
- For rate CSV, `txn_count / total_txn` should match Excel `Balanced Peer Average (%)`.
- Use `utils\csv_validator.py` to confirm for rate outputs.

## 4) Flag-by-flag expectations

Use this section as the acceptance checklist for each case.

### Core inputs
- `--csv`: analysis runs on the specified dataset.
- `--metric` (share): metric must exist in CSV, appears in Summary.
- `--total-col` / `--approved-col` (rate): columns must exist, appear in Summary.
- `--fraud-col` (rate): if present, adds fraud rate output and (if publication) fraud sections.
- `--entity`: target-specific columns are present in dimension sheets.
- no `--entity`: peer-only mode, no target columns.
- `--entity-col`: entity lookup uses the specified column.

### Dimension selection
- `--dimensions`: only those dimensions are analyzed.
- `--auto`: dimensions are auto-detected (logged).
- If neither is set and auto is disabled in config, analysis should fail.

### Time awareness
- `--time-col`: output includes time column; time-aware consistency logs appear.
- Omit `--time-col`: no time column in outputs.

### Output format
- `--output-format analysis`: analysis workbook only.
- `--output-format publication`: publication workbook only (summary + dimension sheets).
- `--output-format both`: both outputs created.
- `--publication-format`: alias for publication format.

### Validation
- default or `--validate-input`: Data Quality sheet exists; warnings logged.
- `--no-validate-input`: no Data Quality sheet.

### Debug / diagnostics
- `--debug`: unweighted averages and weight details included.
- In current defaults, debug/diagnostic sheets are enabled unless config disables them.
- Expected diagnostic sheets when enabled:
  - Peer Weights
  - Privacy Validation
  - Weight Methods
  - Rank Changes

### Presets / config
- `--preset`: Summary references the preset name.
- `--config`: Summary references custom config file.
- `--preset` + `--config`: config merges with preset; overrides visible in Summary.

### Enhanced analysis
- `--compare-presets`: Analysis workbook contains **Preset Comparison** sheet.
- `--analyze-distortion`: Analysis workbook contains **Impact Analysis** sheet.
- `--secondary-metrics`: Analysis workbook contains **Secondary Metrics** sheet.

### CSV export
- `--export-balanced-csv`: creates `<output>_balanced.csv`.
- `--include-calculated`: adds Raw_*, Balanced_*_Share_%, and *_Impact_PP columns.

### Weight strategy
- default: global weights (Weight Methods sheet shows Global-LP).
- `--per-dimension-weights`: Weight Methods sheet shows Per-Dimension methods.

### Advanced optimization
- `--auto-subset-search`: subset search attempts logged; if triggered, Subset Search sheet appears.
- `--subset-search-max-tests`: logged in Summary optimization parameters.
- `--trigger-subset-on-slack`: subset search triggered when slack used.
- `--max-cap-slack`: slack threshold reflected in Summary.

### Rate-specific output
- `--fraud-in-bps`: fraud columns in publication workbook are in bps.
- `--no-fraud-in-bps`: fraud columns remain percentages.

## 5) How to record results

For each case in `cases.jsonl`, record:

- Exit code (0 = success)
- Files created (analysis/publication/csv/audit)
- Any missing/extra sheets vs expectations
- Any errors or warnings not mentioned in README.md

A simple template to track results:

```
Case ID:
Command:
Status:
Observed outputs:
Missing/extra sheets:
Notes:
```

## 6) Known boundaries

- If the CSV lacks a fraud column, fraud-related cases are not generated.
- If the CSV lacks a time column, time-aware expectations do not apply.
- If config defaults are changed, update the expectations accordingly.
